Web Accessibility & Standards

Learn how web accessibility initiatives and standards help ensure inclusivity for all users.

5.1 WAI (Web Accessibility Initiative)

Who

Led by the W3C, the WAI works to improve web accessibility for people with disabilities.

What

WAI provides guidelines like the Web Content Accessibility Guidelines (WCAG) to ensure that websites are accessible. Explore the guidelines on the W3C WAI Website.

Example:

Using alt attributes for images: <img src="example.jpg" alt="Description of the image">

5.2 Section 508 (U.S.)

What

A U.S. law requiring federal websites to be accessible to all citizens. Learn more about Section 508 at the Section 508 Official Site.

Example:

Adding closed captions to a video: <video controls><track src="captions.vtt" kind="captions" srclang="en" label="English"></video>

5.3 ARIA (Accessible Rich Internet Applications)

What

ARIA provides additional markup to improve accessibility for dynamic web content. Read more on the MDN ARIA Guide.

Example:

Using ARIA roles for buttons: <div role="button" tabindex="0" aria-pressed="false">Submit</div>

5.4 Impact on Modern Web Design

What

Inclusive design ensures that websites are accessible to all users. Designers use WCAG guidelines to create flexible, user-friendly interfaces. Explore inclusive design practices on the A11Y Project.

Example:

Responsive design for mobile and desktop: @media (max-width: 768px) { body { font-size: 14px; } }
