3. Web 1.0

3.1 HTML / HTTP Protocol (HTTP Browser/Server). Tim Berners-Lee (Creator)

Who

Tim Berners-Lee is a British computer scientist who invented the World Wide Web. He created the core technologies that made the web possible, including HTML and HTTP.

What

HTML (Hypertext Markup Language) is the standard language for creating documents on the web, and HTTP (Hypertext Transfer Protocol) is the protocol that web browsers use to communicate with web servers.

Why

Berners-Lee developed these technologies to make information sharing on the internet more accessible. Before HTML and HTTP, the internet was primarily used for text-based communication, but these innovations allowed for multimedia-rich web pages and hyperlinks.

When

He created the first version of HTML and HTTP between 1989-1990 while working at CERN. The first web page went live in 1991, marking the birth of the World Wide Web.

How

HTML structures web pages, while HTTP defines how data is exchanged between a browser and a server. This interaction formed the basis of the first websites, enabling the creation of static, text-based pages with hyperlinks.

Example:

The first website, created by Berners-Lee, was an informational page about the World Wide Web project, explaining how users could create and access web pages using HTML and HTTP.

3.2 JavaScript

Who

JavaScript was created by Brendan Eich, a programmer at Netscape Communications Corporation, in 1995.

What

JavaScript is a scripting language that brought interactivity to static web pages during the Web 1.0 era. It allowed developers to add dynamic features like pop-ups, form validation, and basic animations.

Why

JavaScript was introduced to make web pages more interactive without requiring a full reload. Before JavaScript, most websites were static, and any form of user interaction required communication with the server.

When

JavaScript was first released in 1995 as part of Netscape Navigator 2.0, becoming one of the core components of Web 1.0.

How

In Web 1.0, JavaScript was used to dynamically manipulate HTML elements within the browser, allowing users to interact with elements like forms and buttons without reloading the page.

Example:

A common Web 1.0 use case was client-side form validation, where JavaScript would check if a user entered a valid email before sending the data to the server.

What

CSS is a style sheet language used to control the presentation of HTML documents, allowing developers to separate content (HTML) from design (CSS).

Why

Before CSS, all styling was applied directly within HTML, making it difficult to maintain and update websites. CSS enabled developers to define styles separately, reducing code duplication and making website management easier.

When

CSS was first proposed in 1994 and became a W3C recommendation in 1996, though it was not widely adopted until browsers began supporting it in the late 1990s.

How

CSS works by targeting HTML elements and applying visual styles like colors, fonts, and layouts. In Web 1.0, CSS was used mainly for basic design tasks, such as changing fonts or background colors.

Example:

During Web 1.0, CSS was commonly used to style headers (<h1> through <h6>) consistently across a website, applying the same font and color to all headings without manually styling each one in HTML.

3.4 Perl & PHP Server-Side Technologies

Who

Perl was developed by Larry Wall in 1987, and PHP (Personal Home Page, now Hypertext Preprocessor) was created by Rasmus Lerdorf in 1994 for dynamic web development.

What

Perl and PHP are server-side scripting languages that allowed websites to generate dynamic content. These technologies helped move beyond static pages, providing customized responses to user inputs.

Why

Web 1.0 sites were mostly static, and Perl and PHP enabled the creation of dynamic websites, where content could change based on user interactions or data from a database.

When

Perl was widely used in the early 1990s, and PHP was introduced in 1994. Both languages played important roles in the evolution of dynamic web development during Web 1.0.

How

Perl was commonly used in CGI scripts, while PHP allowed server-side logic to be embedded directly in HTML. Both languages generated HTML content dynamically based on user input or server data.

Example:

A Perl CGI script might process a form submission and return a confirmation page, while PHP could pull data from a database to display personalized content based on user preferences.

3.5 Linux

Who

Linux was created by Linus Torvalds, a Finnish computer science student, in 1991 as an open-source operating system kernel.

What

Linux is a Unix-like operating system widely used on web servers during Web 1.0 due to its stability, flexibility, and open-source nature, offering a free alternative to proprietary systems.

Why

Linux's open-source model made it popular among developers, while its stability and security made it ideal for running web servers.

When

Linux was released in 1991, and by the mid-1990s, it had gained popularity as a server operating system, hosting many Web 1.0 websites.

How

Linux was primarily used to run web servers and host websites, often paired with technologies like Apache (web server software), MySQL databases, and scripting languages like PHP and Perl.

Example:

Many early Web 1.0 sites were hosted on Linux servers, using Apache to serve web pages and PHP to dynamically generate content from a database.

3.6 Early Search Engines

Who

Early search engines were developed by academic researchers and companies to help users navigate the growing web. Notable examples include Archie (1990), Lycos (1994), AltaVista (1995), Yahoo! (1995), and HotBot (1996).

What

These search engines indexed web content and allowed users to search for relevant information using keyword queries, helping users find specific content among the expanding web.

Why

As the web grew, users needed a way to find information efficiently. Search engines provided searchable indexes of web pages, helping users locate content more quickly.

When

The first search engine, Archie, appeared in 1990. By the mid-1990s, search engines like Lycos, AltaVista, Yahoo!, and HotBot became widely popular tools for navigating Web 1.0.

How

Early search engines crawled and indexed web pages, ranking them based on relevance, keyword density, and backlinks. Some, like Yahoo!, functioned as directories curated by human editors, while others, like AltaVista, used automated algorithms.

What Happened to Them?

Archie: The first search engine for locating FTP sites. It became obsolete as the web grew. Failed.

Lycos: Initially successful, but struggled to compete with later search engines like Google. Failed in search, survives in other areas.

AltaVista: Once popular, it was acquired by Yahoo! and shut down in 2013. Failed.

Yahoo!: A major player early on, but eventually lost ground to Google. Partially succeeded.

HotBot: Known for speed, but lost relevance with Google's rise. Failed.
